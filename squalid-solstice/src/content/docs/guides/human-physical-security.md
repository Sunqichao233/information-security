# 人的物理セキュリティ対策

本页整理与人员行为、办公现场、门禁控制、组织管理相关的高频安全知识点。

## 1. 社会工程学

社会工程学是不依赖技术漏洞、而是利用人的心理弱点来获取或滥用信息的攻击方式。常见手法包括冒充身份、诱导泄露口令、翻找废弃资料等。

## 2. 肩窥攻击（Shoulder Hacking）

肩窥攻击是社会工程学的一种。攻击者通过在他人身后偷看屏幕或键盘输入，窃取账号、密码或敏感业务信息。

## 3. BEC（商务邮件诈骗）

BEC（Business Email Compromise）指攻击者冒充高管或合作方发送高仿真邮件，诱导员工转账到伪造账户，或套取内部机密信息。

## 4. 内部控制不完善

内部控制不完善是指组织用于预防舞弊与失误的制度未建立，或建立后执行不到位，导致管理失效、风险上升。

## 5. 影子 IT（Shadow IT）

影子 IT 指员工或部门在未纳入统一 IT 管理情况下，自行引入系统、云服务或设备。由于绕过组织安全策略，容易成为数据泄露和恶意软件入侵路径。

## 6. 安全审查（Security Clearance）

安全审查是对可接触高敏感或国家安全相关信息人员进行资格认定与持续审查的制度，用于控制关键机密信息的访问范围。

## 7. 保密协议/誓约书（NDA）

NDA（Non-Disclosure Agreement）用于约束员工或合作方的保密义务，通常规定信息使用范围、违规责任，并可能约定离职/合同终止后的持续保密期限。

## 8. RASIS 可靠性评价指标

RASIS 是评估系统可信性的五维框架：

- Reliability（可靠性）：系统持续稳定运行能力，常用指标为 MTBF。
- Availability（可用性）：系统在需要时可被使用的程度，常用指标为稼动率。
- Serviceability（可维护性）：系统故障后修复与恢复的容易程度，常用指标为 MTTR。
- Integrity（完整性/保全性）：数据保持正确、一致、不被破坏的能力。
- Security（安全性）：防止未授权访问、篡改、泄露的能力。

## 9. RAS 技术

RAS 技术通常指 Reliability、Availability、Serviceability 三项，强调“少故障、不断机、易恢复”的系统运维目标。

## 10. 安全分区（Security Zoning）

安全分区是按信息敏感级别划分区域并设置不同准入控制，仅允许具有对应权限的人员进入相应区域，实现分级隔离与最小权限。

## 11. 反尾随（Anti-passback）

反尾随用于防止未认证人员紧跟已认证人员进入受控区域。系统通过记录出入状态与认证顺序，识别并阻止违规通行。

## 12. 联锁（Interlock）

联锁是一种“前置条件满足后才允许下一动作”的控制机制。门禁场景中常见双门互锁，一次只允许一人通过，以抑制尾随入侵。

## 13. 清桌面与清屏（Clear Desk / Clear Screen）

- 清桌面：离开工位前，不在桌面遗留敏感纸质资料、移动存储介质等。
- 清屏：离席立即锁屏，防止他人窥视或未经授权操作终端。

## 14. USB Key（安全U盘密钥）

USB Key 是 USB 形态的安全密钥设备，可用于登录认证、数字签名、数据加解密等场景。通过“持有实物密钥 + 凭证”可提升访问控制强度。
