# 标的型攻击、DoS攻击、钓鱼

## 标的型攻击（Targeted Attack）

针对特定组织、部门或个人进行的定向攻击。常见方式是先进行信息收集，再通过邮件、社工或恶意文档等手段实施入侵。

其典型特征是：会提前调查目标系统与组织的脆弱点，进行周密准备，并组合多种攻击手段实施入侵。部分攻击还会使用专用恶意软件（变种），因此较难通过单一措施彻底防住。

### APT（Advanced Persistent Threat）

APT 常译为“高级持续性威胁”，指面向特定目标、长期持续推进的网络攻击活动。

这类攻击往往组织化程度高、隐蔽性强，常见于高价值目标场景，可视为标的型攻击中的典型形态之一。

### 标的型攻击邮件（Spear Phishing Mail）

这是针对特定企业或组织的邮件攻击形式。攻击者会伪装成目标信任的人物或相关方，并使用与目标业务/关系高度相关的话题诱导受害者点击链接、打开附件或执行操作。

由于常使用定制化手法与恶意程序变种，这类邮件在早期往往更难被通用规则直接检出。

### 往来型攻击（Thread Hijacking / 会话伪装）

攻击者先与目标通过邮件等方式进行多次看似无害的交流，待建立一定信任关系后，再投递恶意内容实施攻击。

其危险点在于“先建立信任再攻击”，比一次性钓鱼更具迷惑性。

### 水坑攻击（Watering Hole Attack）

攻击者预先在目标用户“高频访问的网站”上布置恶意代码或陷阱，等待目标访问后触发攻击。

该方式属于标的型攻击的一种，重点在于“蹲守目标常去的网站”。

### 网络攻击杀伤链（Cyber Kill Chain）

“杀伤链”是从攻击者视角对标的型网络攻击流程的阶段化模型。常见为 7 个阶段，防守方只要在任一阶段有效阻断，都可能打断后续攻击。

7 个阶段可概括为：

1. 侦察（Reconnaissance）
2. 武器化（Weaponization）
3. 投递（Delivery）
4. 利用/执行（Exploitation）
5. 安装（Installation）
6. 命令与控制（C2）
7. 目标达成（Actions on Objectives）

## 标的型攻击的对策

标的型攻击通常难以通过单一技术手段完全阻断，因此需要持续实施组织级防护措施。

- 收到可疑邮件时，第一时间上报并共享给安全窗口/安全团队。
- 持续开展员工安全教育与演练，提升识别可疑邮件与异常行为的能力。

## DoS 攻击（Denial of Service）

DoS 是 `Denial of Service` 的缩写，意为“拒绝服务”。

攻击者通过大量请求、海量邮件或异常处理负载，使服务器处理能力被耗尽，导致服务降级、变慢、宕机或不稳定。

### DDoS（Distributed DoS）

DDoS 中的 `D` 指 `Distributed`（分布式）。

其特点是由多个终端/节点同时发起 DoS 攻击，攻击流量更大、来源更分散，通常更难快速溯源与拦截。

### EDoS（Economic Denial of Sustainability）

EDoS 常见于使用云服务并按流量/资源用量计费的网站场景。

攻击者通过持续制造过量流量或请求，诱导云资源扩容并放大计费，进而造成显著经济损失；严重时可导致业务难以持续运营。

### SYN Flood 攻击

SYN Flood 是 DoS 攻击的一种。攻击者向目标持续发送大量 TCP 连接初始请求（SYN 包），占用连接队列与系统资源，导致正常连接难以建立。

### Smurf 攻击

Smurf 也是 DoS 攻击的一种。攻击者伪造“目标主机 IP”作为源地址，向广播地址发送 ICMP 请求，使大量回复流量集中回到受害者，从而压垮目标网络或主机。

## DoS / DDoS 攻击对策

常见防护手段包括部署与联动：

- WAF（Web Application Firewall）
- IDS（Intrusion Detection System，不正入侵检测系统）
- IPS（Intrusion Prevention System，不正入侵防御系统）

此外，在部分 DoS 场景下，对恶意来源地址实施 IP 封禁也可能有效；但面对分布式且伪造来源的大规模攻击时，通常需要结合上游清洗与多层防护策略。

## DNS 缓存投毒（DNS Cache Poisoning）

攻击者篡改 DNS 服务器缓存中的“域名与 IP 地址映射关系”，使用户在访问正常域名时被错误解析到恶意地址。

### DNS 缓存投毒对策

DNS 解析服务应尽量只响应内部递归查询（来自受信任网段/客户端的递归请求），不对外部来源开放递归查询能力，以降低被投毒利用的风险。

## SEO 投毒（SEO Poisoning）

攻击者利用搜索引擎优化机制或算法弱点，让恶意网站在搜索结果中靠前展示，从而诱导用户进入钓鱼站点或恶意软件下载页面。

## 钓鱼攻击（Phishing）

“钓鱼（Phishing）”是指攻击者通过邮件等渠道把受害者引导到伪造网站，诱导其输入 ID、密码等敏感信息，从而实施信息窃取。

### 短信钓鱼（Smishing）

Smishing 是 `SMS + Phishing` 的合成词，指主要通过短信（SMS）实施的钓鱼诈骗。

常见形式是发送伪装通知链接，引导用户在假页面提交账号、验证码或支付信息。

## MITB 攻击（Man in the Browser）

MITB 是“浏览器中的中间人”攻击。攻击者先控制受害者浏览器，再对浏览器与服务器之间的通信进行窃听或篡改。

这类攻击可在用户看似正常访问网银等站点时插入伪造页面或篡改交易内容，常被视为中间人攻击的一种变体。

## 中间人攻击（MITM, Man in the Middle）

MITM 指攻击者插入到通信双方之间，伪装成彼此的通信对象，在不易被察觉的情况下实施窃听、篡改或重放数据。

与 MITB 的区别在于：MITB 更强调“浏览器端被劫持”；MITM 是更广义的“通信链路中间劫持”。

## 会话劫持（Session Hijacking）

会话是指通信双方持续交换数据的连接状态，而会话 ID 用于标识该会话。

会话劫持是指攻击者通过窃取或猜测会话 ID，冒充通信中的一方继续与另一方交互，从而实施数据窃取、越权操作等行为。

## IP 欺骗（IP Spoofing）

IP 欺骗是指篡改数据包中的源 IP 地址字段，伪装发送方身份的行为。

该手法常被用于隐藏攻击来源、配合反射/放大型攻击，或绕过基于来源地址的简单访问控制。

## 点击劫持（Clickjacking）

点击劫持是指攻击者通过界面欺骗，让用户在不知情情况下点击透明或伪装的页面元素，从而触发非本人意图的操作。

常见方式包括把恶意按钮覆盖在正常页面上，诱导用户误点。

## Evil Twin（双胞胎热点攻击）

攻击者在公共 Wi-Fi 附近搭建与合法热点名称相似的伪造接入点，诱导用户误连后，对其通信进行监听或中间人攻击。

## 冒充类攻击的对策

### 钓鱼攻击对策

- 启用多因素认证（即使口令泄露也不易直接登录）。
- 不随意点击邮件中的 URL，优先通过官方入口自行访问。
- 启用并验证发件人认证机制（如 SPF/DKIM/DMARC）。

### 中间人攻击（MITM）对策

- 通过数字证书校验通信对端身份，避免连接到伪造站点。

### MITB 攻击对策

- 在关键交易中引入交易签名/交易确认机制，防止页面被篡改后静默提交。

#### 交易签名（Transaction Signing）

交易签名是指：用户对其已确认的交易内容（如收款方、金额等）进行签名，由服务器验证签名，以同时确认“操作者身份”与“交易内容完整性”。

其关键在于“把具体交易内容一并纳入签名”，从而防止交易在传输或浏览器侧被悄悄篡改。

常见实现方式包括：

- 使用 IC 卡/U 盾等硬件令牌对交易内容签名。
- 使用手机 App 等独立通道设备确认交易内容并完成签名。

### 会话劫持对策

- 对高风险操作要求再次输入口令或进行二次认证。
- 全程使用 TLS（Web 场景使用 HTTPS）。
- 提高会话 ID 强度与随机性，降低被猜测或伪造概率。
